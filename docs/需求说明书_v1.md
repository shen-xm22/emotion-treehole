## 一、项目概述

### 1.1 一句话定位

> **一个让用户愿意反复回来的匿名情绪空间——做测评、写心情、跟AI树洞聊一聊。**

### 1.2 核心价值

| 解决什么问题 | 怎么解决 |
|:-----------|:--------|
| 情绪没有出口 | 匿名、无社交压力、随时可倾诉的树洞 |
| 不了解自己 | 专业心理学简易测评 + AI深度解读 |
| 负面情绪积累 | 每日打卡 + 情绪趋势可视化 |
| 孤独感 | 树洞记住你、了解你、持续陪伴 |

### 1.3 目标用户

- 18-35岁，城市青年
- 有轻度到中度情绪困扰（焦虑、内耗、人际关系烦恼）
- 不想/不方便找心理咨询师
- 需要一个安全、低门槛的情绪出口

## 二、产品功能

### 2.1 功能全景图

```
树洞情绪站
├── 访客模式（无需登录）
│   └── 三个测评（性格/焦虑/关系）
├── 注册用户
│   ├── 测评系统
│   │   ├── 性格风格（大五人格简化版）
│   │   ├── 焦虑评级（改良GAD-7）
│   │   └── 亲密关系（依恋理论）
│   │   └── 测评报告（基础评分 + AI深度解读）
│   ├── 树洞聊天
│   │   ├── 自由对话（AI智能倾听）
│   │   ├── 会话记忆（跨对话回访引用）
│   │   └── 危机转介（识别高危情况）
│   ├── 情绪记录
│   │   ├── 每日打卡（1-10分 + 关键词）
│   │   └── 情绪日历（月视图热力图）
│   │   └── 趋势报告（周/月变化分析）
│   └── 个人中心
│       ├── 测评历史
│       ├── 对话记录
│       └── 情绪档案
└── 付费增值
    ├── AI深度解读报告（¥9.9/次）
    ├── 情绪树洞解锁（¥9.9/次）
    └── 月趋势报告（¥19.9/月）
```

### 2.2 测评模块

三个标准化量表，但交互要轻量、视觉要治愈。

| 测评 | 题量 | 参考理论 | 输出 |
|:----|:----|:--------|:----|
| 🧩 性格风格 | 12题 | Big Five简化 | 五维雷达图 + 性格画像描述 |
| 😰 焦虑评级 | 10题 | GAD-7改良 | 焦虑分数 + 等级标签 + 高危预警 |
| 💕 亲密关系 | 15题 | 依恋理论 | 依恋类型 + 关系模式解读 |

**互**：每题一屏，滑屏/点选，进度条显示进度。
**报告**：评分 + 可视化图 + AI生成300-1500字个性化解读。

### 2.3 树洞聊天模块 ⭐ 核心功能

这是产品的灵魂。实现要点：

| 维度 | 说明 |
|:----|:-----|
| 对话AI | DeepSeek API + 树洞人设系统提示 |
| 上下文 | 当前对话最近30条消息 |
| 跨会话记忆 | 每次对话后AI生成总结，下次注入 |
| 用户画像 | 测评结果 + 关键信息提取 → 注入上下文 |
| 流式输出 | SSE流式响应，逐字显示（打字机效果） |
| 敏感检测 | 识别自伤/自杀/暴力倾向 → 触发转介话术 |

**人设详见**：`docs/树洞人设_v1.md`

### 2.4 情绪记录模块

| 功能 | 说明 |
|:----|:-----|
| 日打卡 | 今天心情几分（1-10）+ 选一个关键词（可选多个） |
| 关键词库 | 预设20-30个情绪标签（焦虑/平静/开心/疲惫等） |
| 日历视图 | 月历热力图，颜色深浅表示情绪值 |
| 趋势报告 | 周报/月报：平均分、波动幅度、关键词频率、AI趋势解读（付费） |

### 2.5 个人中心

- 测评历史记录
- 树洞对话回顾（树洞的自动总结版，非全文）
- 情绪档案汇总

## 三、AI记忆系统（核心架构设计）

### 3.1 记忆模型

```
用户
├── Profile（固定信息）
│   ├── 测评结果（各维度分数 + 类型标签）
│   └── 用户偏好的称呼/表达方式
├── Memory Snapshots（每次对话后生成）
│   ├── 时间戳
│   ├── 情绪摘要（2-3句话）
│   ├── 关键事实提取（人物/事件/情境）
│   └── 当前情绪评分
├── Conversation Logs（原始对话记录）
│   └── 每轮：用户消息 + AI回复 + 时间
└── Emotion Check-ins（情绪打卡）
    ├── 每日评分 + 关键词
    └── 打卡时间
```

### 3.2 记忆工作流程

```
用户发送消息 → 检索上下文
    ├── 当前对话历史（最近30条）
    ├── 最近的3条 Memory Snapshot
    ├── 用户 Profile（测评结果 + 关键事实）
    └── 最近7天情绪打卡
        ↓
组装成完整上下文 → 发送给 DeepSeek → 回复用户

对话结束后（用户离开/空闲超时）
    ↓
AI 自动生成新的 Memory Snapshot
    ├── 本次对话的情绪摘要
    ├── 新发现的关键事实
    └── 观察到的情绪变化
        ↓
存入数据库
```

### 3.3 记忆召回示例

| 场景 | 树洞表现 |
|:----|:--------|
| 第二天回来 | 「昨天说的工作烦心事，今天好点了吗？」 |
| 一周后回来 | 「上次你说在和妈妈冷战——后来怎么样了？」 |
| 测评后第一次聊天 | 「我看到你的测评结果了……你对自己还挺严格的，但不用太担心」 |
| 连续低落一周 | 「这周你连续几天打分都不高……要不要聊聊发生了什么？」 |

## 四、技术架构

### 4.1 部署架构

```
用户浏览器（手机/电脑）
    │
    ▼
云服务器（阿里云/腾讯云轻量）
    ├── Nginx（反向代理 + 静态文件 + SSL）
    ├── FastAPI + Uvicorn（Web后端）
    ├── PostgreSQL / SQLite（数据库）
    └── Redis（可选，会话缓存 + 速率限制）
        │
        ▼
DeepSeek API（OpenAI兼容接口）
```

### 4.2 技术选型

| 层 | 技术 | 选型理由 |
|:--|:----|:--------|
| 前端 | HTML + CSS + 原生JS | 轻量、无需构建、可内嵌 |
| 后端 | Python FastAPI | 开发快、天然支持异步API和SSE |
| 数据库 | SQLite起步 → PostgreSQL | 小规模够用，上云可平滑迁移 |
| AI | DeepSeek API（已有key） | 性价比高，支持长上下文 |
| 部署 | 阿里云/腾讯云轻量服务器 | 2C4G足够，年费几百 |

### 4.3 数据模型

```
用户表 (users)
├── id (主键)
├── nickname (自动生成)
├── created_at
└── last_active_at

测评结果表 (assessments)
├── id | user_id | type (性格/焦虑/关系)
├── scores (JSON：各维度分数)
├── report_text (AI生成的解读)
└── created_at

对话记录表 (conversations)
├── id | user_id | session_id
└── created_at

消息表 (messages)
├── id | conversation_id | role (user/assistant)
├── content (消息内容)
└── created_at

记忆快照表 (memory_snapshots)
├── id | user_id
├── summary (情绪摘要)
├── key_facts (JSON：提取的关键信息)
└── created_at

情绪打卡表 (mood_checkins)
├── id | user_id | score (1-10)
├── tags (JSON：关键词列表)
└── created_at
```

### 8.1 竞品参照

| 产品 | 优点 | 不足 |
|:---|:----|:----|
| 壹心理 | 专业深度 | 重、商业化强、不够轻量 |

### 8.2 风险与对策

| 风险 | 概率 | 对策 |
|:---|:---:|:----|
| AI回复质量不行 | 中 | 持续优化人设提示 + 评测机制 |
